# 项目计划：Entering *Las Meninas*: How a Painting Changed the Way I Look at Art

> 课程：CogSci 139 — Art, Geometry, and Cognition
> 当前状态：计划已执行，正式结果与交付物已冻结
> 本文目的：保留研究问题、创作方向、CNN 方法与实验变量的预注册式设计记录。实际运行结果见 [`report/final-report-zh.md`](report/final-report-zh.md)、[`outputs/metrics/summary.json`](outputs/metrics/summary.json) 与项目首页。

内部副标题：**The CNN as an Incomplete Viewer: Separating Style, Geometry, and Relational Topology**

## 1. 项目摘要

本项目研究委拉斯开兹的《宫娥》（*Las Meninas*）如何通过透视、镜像、遮挡、人物位置与凝视关系，把画外观看者纳入作品内部。项目不会把神经网络仅仅当成一个生成滤镜，而会把 CNN 当作一种“不完整的观看者”：它可以提取颜色、纹理、局部轮廓、人物和空间层次，却未必能理解镜子、画中画、画外人物及观看者位置共同形成的意义。

项目的核心问题是：

> **CNN 能否把《宫娥》的风格、几何结构和关系拓扑分开？其中哪一层真正让观看者产生“进入画中”的感觉？**

项目计划对原作进行多组受控 transformation，分别改变风格、几何和关键关系，再比较：

1. CNN 的特征表示把哪些版本判断为更接近原作；
2. 人类观看者认为哪些版本仍然是《宫娥》；
3. 哪些变化会削弱或破坏“我也在画中”的感受；
4. 人类和 CNN 的判断在哪些地方出现分歧。

最终成果既可以是一组经过严格控制的变体，也可以在这些实验基础上发展出一件个人化的新作品。

## 3. 为什么选择《宫娥》

《宫娥》适合本课程，不只是因为它是一幅著名绘画，而是因为作品不断迫使观看者建立一张空间和关系的 cognitive map。

观看者需要推断：

- 委拉斯开兹正在画谁；
- 镜中出现的人物位于哪里；
- 国王与王后是否与现实观看者占据相近的位置；
- 画家、Infanta、侍女、镜子、后门与大画布之间是什么关系；
- 画面中的人物是在看国王王后、观看者，还是画外某个并不存在于图像中的对象；
- 后门、镜框和大画布分别通向现实空间、反射空间还是再现空间。

这意味着作品的意义不能被简化为“识别出了多少人物”。人类会把局部视觉线索组织成一个包含画外空间的整体模型，而 CNN 可能只保留某些视觉统计与局部模式。两者之间的差异正是本项目的认知科学价值。

## 4. 核心概念框架：三个可分离但互相作用的层次

| 层次 | 本项目中的定义 | 可能的 CNN 表示 | 典型问题 |
| --- | --- | --- | --- |
| Style | 色彩、明暗、局部纹理、笔触和表面外观 | 浅层/中层 activation 与 Gram matrix | 外观改变后，它仍然是《宫娥》吗？ |
| Geometry | 位置、距离、尺度、角度、消失点、透视和深度 | 保留空间坐标的 feature map、深度图、分割结果 | 空间结构改变后，观看者还会感到自己在画中吗？ |
| Relational topology | 元素之间是否存在某种定性关系，如反射、遮挡、包围、前后与凝视 | 分割区域加关系图；CNN 结果必须由人进行审查 | 即使风格和整体构图相近，关键关系消失后，作品意义是否改变？ |

### 4.1 Style 不是“真正的艺术风格”

CNN 的 style descriptor 通常更接近颜色与纹理统计，而不是完整的艺术史风格。因此，文中应使用“CNN 提取的风格表征”或“机器定义的风格距离”，不能声称模型已经理解了委拉斯开兹的艺术风格。

### 4.2 Geometry 必须保留空间位置

如果把 CNN 特征直接做全局平均池化，精确的空间关系会被削弱。因此，几何分析需要使用仍然保留高度和宽度维度的 feature map，必要时加入专门的深度、分割或线条检测 CNN。

### 4.3 Relational topology 是操作性定义

本项目中的关系拓扑指元素之间的定性连接关系，而不是严格证明得到的神经流形，也不是人脑拓扑的直接测量。例如：

- 镜子反射画外人物；
- 后门包围背景人物；
- 大画布遮挡左侧空间；
- 画家和多个人物面向观看者；
- Infanta 位于人物群的视觉中心；
- 观看者的位置与画外国王王后的位置发生概念重叠。

项目必须把这些关系明确列出，才能讨论 transformation 是否保留或破坏了作品的关系结构。

## 5. 研究问题与初步假设

### 5.1 研究问题

**RQ1：** CNN 的不同层是否会对风格变化和几何变化表现出不同敏感度？

**RQ2：** 仅改变颜色、纹理和笔触时，“进入画中”的感觉能否被保留？

**RQ3：** 改变消失点、房间深度、镜子或后门的位置时，观看者的空间认知会如何变化？

**RQ4：** 当镜像、凝视或画外人物关系被破坏时，CNN 与人类是否会对作品相似性给出不同判断？

**RQ5：** 毕加索版本在风格和几何上都与原作相距很远，人类为何仍能认出它与《宫娥》的关系？CNN 是否也能保持这种跨变形的识别能力？

### 5.2 初步假设

**H1：** 只改变风格会显著改变 CNN 的浅层纹理统计，但不会立即破坏人类观看者的“进入感”。

**H2：** 改变消失点、纵深和人物相对位置会明显削弱空间进入感，即使画面的颜色和笔触仍接近原作。

**H3：** 删除镜子、关闭后门或改变人物凝视等关系拓扑后，作品的意义会发生比普通风格转换更大的改变。

**H4：** CNN 可能认为一个纹理相似但关系被破坏的版本更接近原作，而人类可能认为风格和几何都发生巨大变化的毕加索版本更接近原作。这种分歧可以体现机器视觉与人类泛化之间的差异。

这些都是待检验假设，不能在获得结果之前写成已经成立的结论。

## 7. 总体实验设计

### 7.1 2 × 2 transformation 矩阵

|  | 风格保持 | 风格改变 |
| --- | --- | --- |
| 几何保持 | **G0S0：原作或最小处理基线** | **G0S1：保持人物与构图，只改变颜色、纹理和笔触** |
| 几何改变 | **G1S0：保持原作外观，改变消失点、纵深、门或镜子** | **G1S1：同时改变几何和风格，形成个人最终作品** |

这个矩阵的意义是让风格和几何成为可控制变量，而不是让生成模型一次性改变所有内容。

### 7.2 增加关系拓扑破坏实验

在 2 × 2 矩阵之外，增加至少一个 **T1** 版本：尽量保持整体颜色、纹理和基本构图，但单独破坏一个关键关系，例如：

- 删除镜子；
- 关闭后门；
- 让主要人物不再面向观看者；
- 删除左侧大画布；
- 让镜中人物变成画内人物，取消画内与画外的连接。

这一组实验用于判断关系变化是否会造成超出普通 style/geometry distance 的认知影响。

为了降低“只是局部颜色、边缘或面积发生变化”造成的混淆，关系实验需要一个 matched control。例如，若 `T1` 删除镜中人物，则对照版本应使用亮度、边框与面积相近的普通画面替换镜中区域。这样才能更有把握地把差异归因于反射关系，而不是简单的像素变化。

### 7.3 使用连续强度而不是只有前后对比

对于最重要的一项 transformation，建议设置多个强度，例如：

- 0%：原作；
- 25%：轻度改变；
- 50%：中度改变；
- 75%：高度改变；
- 100%：完整变换。

连续序列可以形成 CNN feature space 中的轨迹，也能找到“观看体验发生突变”的大致位置。它比仅比较原图和一张终图更有分析价值。

## 8. CNN 方法设计

### 8.1 核心原则

CNN 不是用来完成一个过于简单的“是否为《宫娥》”分类任务，而是作为多层视觉测量工具。项目需要保留和解释不同层的 activation，并让每种 measurement 对应明确的研究问题。

### 8.2 预训练 CNN 主干

初步候选是易于解释、适合提取多层特征的预训练 CNN，例如 VGG 或 ResNet。最终选择需要根据以下标准决定：

- 能否方便访问浅层、中层和深层 activation；
- 是否容易保留二维 feature map；
- 是否适合计算 style 与 content/geometry distance；
- 是否能在本地 Apple Silicon 环境稳定运行；
- 是否能清楚解释，而不是只追求模型规模。

如果项目采用 CNN neural style transfer，VGG 类型的网络会更加直接；如果更重视语义和鲁棒特征，ResNet 可以作为对照。当前计划不需要同时堆叠多个大模型。

当前推荐的最小技术基线是：

1. 冻结的预训练 VGG，用于多层 activation、Gram style proxy 和保留空间位置的 content feature；
2. HED 一类全卷积边缘 CNN，用于提取绘画中的多尺度边界；
3. Hough 或 RANSAC，根据 CNN 边缘结果估计主要直线与消失方向；
4. Canny 加 Hough 作为非神经网络对照，检查 HED 的输出是否只是模型或参数造成的假象；
5. 全画面工作尺寸与镜子、后门、人物群固定局部 crop 同时分析，避免缩小后关键区域消失。

这套组合让“CNN 提取几何特征”拥有明确含义，同时保留传统几何方法作为验证工具。最终仍需先进行最小可行测试，再确认具体网络、层和输入尺寸。

### 8.3 Style extraction

设第 `l` 层特征为：

```text
F_l(x) ∈ R^(C_l × H_l × W_l)
```

在保留原始 activation 的同时，可以把空间位置展开，计算通道之间的 Gram matrix：

```text
G_l(x) = F_l(x) · F_l(x)^T / (C_l × H_l × W_l)
```

Gram matrix 会弱化特征出现的具体位置，更强调哪些纹理和局部模式共同出现，因此可作为机器定义的 style descriptor。

计划输出：

- 多个浅层/中层 activation map；
- 原作与各变体之间的 style distance；
- style-only transformation 的强度轨迹；
- 对模型是否过度依赖颜色和纹理的解释。

### 8.4 Geometry extraction

几何信息不能只依赖 Gram matrix，因为 Gram matrix 有意忽略空间布局。几何分支计划使用：

- 保留二维位置的中层或深层 feature map；
- 原图与变体之间的 spatial feature correspondence；
- 人物、镜子、门、画布和狗的分割区域；
- 相对深度图；
- 消失点、地平线和主要透视方向；
- 关键区域的中心、面积、尺度和相对位置。

其中至少有一项核心空间表征应来自 CNN，例如 CNN depth estimation、CNN segmentation 或 CNN line detection。传统计算机视觉可以辅助计算消失点或直线交点，但不能把传统算法的全部输出包装成“CNN 提取的特征”。

计划输出：

- 几何 feature map；
- 相对深度可视化；
- 原作与变体的关键区域位移；
- 消失点或纵深变化；
- geometry distance 随 transformation 强度变化的曲线。

### 8.5 Relational topology map

计划把关键元素表示为节点，把关系表示为边。例如：

```text
viewer ↔ reflected-by ↔ mirror
painter → looks-toward → viewer
large-canvas → occludes → left-room
door-frame → contains → background-figure
Infanta → surrounded-by → attendants
```

CNN 可以帮助发现或分割节点，但“镜像”“凝视”“画外人物”之类的关系可能无法被可靠自动识别。模型失败时不应手工伪造一个成功结果，而应明确记录：CNN 看到了哪些视觉区域，又缺失了哪些关系。这个失败本身可以成为项目论点。

### 8.6 CNN feature space 与 topological idea

每个图像版本都可以被表示为 CNN feature space 中的一个点。对连续变换序列，可以观察它们形成的轨迹：

- style-only 轨迹主要改变浅层和 Gram 特征；
- geometry-only 轨迹主要改变保留空间布局的特征；
- topology-breaking 版本可能在 CNN 距离上变化不大，却在人类判断上发生明显变化；
- 毕加索版本可能在像素和 style distance 上很远，但在语义或关系层仍与原作相连。

为了展示这些关系，可以选择 PCA 或 UMAP 进行二维可视化。不过二维降维图只是高维特征的近似投影，不能被写成神经网络或人脑真实结构的证明。

主分析图应优先选择信息更直接的形式：

- `layer × image` 的表示距离热图；
- style distance 与 spatial-content distance 的二维散点图；
- CNN 边缘、检测直线、人工参考消失点三层叠图。

PCA/UMAP 仅作为扩展，不应承担主要论证。

## 9. Transformation 方案

### 9.1 Style-only transformation

目标：尽量保持人物位置、消失点、门、镜子和画布布局，只改变表面外观。

候选变化：

- 改变色彩与明暗分布；
- 使用自制纹理或来源清楚的风格参考；
- 将笔触变得更平面、更几何或更接近个人绘画方式；
- 通过 neural style transfer 保留 content feature、改变 style feature。

验收标准：主要几何 landmark 的位置基本不变；CNN style distance 明显上升；geometry distance 相对较小。

### 9.2 Geometry-only transformation

目标：尽量保留原作色彩和局部纹理，只改变空间组织。

候选变化：

- 压缩或夸张房间纵深；
- 移动消失点；
- 移动或改变后门大小；
- 放大、缩小或移动镜子；
- 改变人物之间的相对距离；
- 把画面从强透视逐渐变为较平面的构图。

验收标准：style distance 相对可控；空间 landmark、相对深度或 feature correspondence 发生明确变化；每个版本只改变预先声明的变量。

### 9.3 Topology-breaking transformation

目标：检验作品意义是否依赖那些无法被普通纹理或位置距离充分描述的关系。

优先候选：

1. 删除镜中人物但保留镜框；
2. 关闭后门但保留其位置；
3. 保持人物位置不变，只改变凝视方向；
4. 删除大画布，使画家不再与“正在创作什么”发生联系。

验收标准：图像整体外观仍然接近原作，但至少一条预先定义的关系边被明确删除或改变。

### 9.4 最终个人作品

最终个人作品不应只是所有效果的随机叠加，而应回答研究问题。一个较强的方向是：

> 保留《宫娥》中“观看者被纳入画面”的关系拓扑，同时使用新的风格和空间几何，使观看者真正成为镜中或画外对象。

作品可以选择以下形式之一：

- 一张完成度高的静态图；
- 一组四幅或五幅的 transformation 序列；
- 一段由 CNN 深度图驱动的 2.5D “进入画中”短动画；
- 一个把现实镜子或摄像头影像放入镜框区域的数字作品；
- 静态作品加特征可视化与分析图组成的展示板。

最终形式应等看到第一轮 CNN 结果后再决定，避免创作形式反过来支配研究问题。

## 10. 评估方法

### 10.1 机器侧指标

| 指标 | 用途 | 局限 |
| --- | --- | --- |
| Style loss / Gram distance | 比较颜色与纹理统计 | 不能代表完整艺术风格 |
| Spatial feature distance | 比较保留空间坐标的 CNN 特征 | 可能同时受到风格变化影响 |
| Feature correspondence | 比较人物和关键区域是否仍可对应 | 对大幅生成变化可能不稳定 |
| Depth-map correlation | 比较空间层次与纵深 | 绘画不是照片，深度模型可能误判镜子和画布 |
| Landmark displacement | 量化门、镜子、人物、消失点的移动 | landmark 可能需要人工复核 |
| Deep semantic similarity | 判断不同版本的高层语义接近程度 | 不能证明模型理解了叙事关系 |

风格距离应尽量逐层报告，而不是只给一个总分。几何位移可除以图像对角线进行归一化，使不同工作尺寸之间能够比较。LPIPS 等感知距离如果使用，只能作为总体差异的辅助指标，不能被称为纯 style 或纯 geometry 指标。

### 10.2 关系拓扑指标

先建立一张小型关系清单，然后记录每个变体保留了多少条关系。例如：

| 关系 | 原作 | 变体是否保留 |
| --- | --- | --- |
| 镜框中存在画外人物的反射 | 是 | 是/否 |
| 画家面对画外观看位置 | 是 | 是/否 |
| 后门中存在背景人物 | 是 | 是/否 |
| 大画布遮挡左侧空间 | 是 | 是/否 |
| Infanta 处于人物群视觉中心 | 是 | 是/否 |

该指标是人为定义并审查的关系表，不伪装成自动得到的客观真理。

### 10.3 人类观看者的小型评价

如果时间允许，可以邀请少量同学对经过随机排序的图像进行 1–5 分评价：

- 我感觉自己位于画面所构造的空间中；
- 我感觉画中人物正在看向我；
- 我仍然认为这件作品保留了《宫娥》的身份；
- 我能清楚理解镜子、门和画外人物的关系。

样本量较小时，结果只能作为探索性观察，不能推广成人类视觉的一般规律。

## 11. 毕加索版本的角色

`picasso.png` 不用于从零训练网络，而作为一个外部比较案例：它同时改变了颜色、笔触、比例、构图和空间几何，但人类仍然能够识别它与《宫娥》的联系。

它可以帮助回答：

- CNN 的浅层特征是否因外观变化而认为两者完全无关；
- 深层特征能否对应画家、Infanta、门与人物群；
- 人类是否依靠角色关系和构图拓扑完成了 CNN 难以实现的泛化；
- 毕加索保留了哪些关系，又主动破坏了哪些几何规则。

项目不应把毕加索版本简单称为“style image”，因为它改变的不只是 style，也改变了 geometry。它更适合充当测试项目框架是否有效的复杂案例。

## 12. Ready-made 的候选方向

如果本学期仍然要求 ready-made，最自然的选择是一面普通镜子，作品名可以暂定为：

> **The Missing Viewer**

镜子不需要被复杂加工。它的意义来自观看者站在镜前时，暂时占据《宫娥》中画外人物的位置。后续可以把它与数字作品结合：

- 在最终图像的镜框区域放入现实观看者；
- 把镜子放在投影或屏幕前，使现实空间与画内空间发生连接；
- 用普通摄像头或手机屏幕作为数字时代的 ready-made。

这个部分必须以老师对本学期 ready-made 要求的最终说明为准。

## 15. 最小方案与扩展方案

### 15.1 最小但完整的版本

- 一个预训练 CNN 主干；
- 原作加三类受控变体；
- style distance 与 spatial feature distance；
- 一张 2 × 2 作品矩阵；
- 一个关系拓扑破坏实验；
- 一件最终个人作品；
- 清楚解释机器表征与人类理解的差异。

这已经能够完整回答课程问题，不需要为了显得高级而堆叠大量模型。

### 15.2 有时间再加入的扩展

- CNN 深度估计与 2.5D 动画；
- PCA/UMAP feature-space trajectory；
- 少量人类观看评价；
- 与毕加索版本的局部 feature correspondence；
- 小型 convolutional autoencoder；
- 摄像头或现实镜子参与的互动展示。

如果老师明确要求亲自训练 CNN，可从原作生成带有已知 transformation 类型和强度的受控样本，训练一个小型多任务 CNN 预测：

- 变化主要属于 style、geometry 还是 topology intervention；
- transformation 的强度等级。

测试集需要按空间区域或 transformation 组合留出，避免把同源随机 patch 泄漏到训练集和测试集。即使模型准确率较高，也只能声称它学习了这件作品内部的变换规律，不能宣称对艺术作品具有普遍泛化能力。

扩展项只有在主实验完成并且结果可解释之后才加入。

## 16. 实施阶段与验收标准

### 阶段一：冻结研究设计

任务：确定最终研究问题、三个 transformation 和是否包含 ready-made。
验收：每个变体只能改变预先声明的主要变量；能够明确解释为什么要做。

### 阶段二：准备输入图像

任务：保存高清原图不变，生成规范命名的工作代理图和必要局部裁切。
验收：尺寸、色彩空间、归一化方式和来源记录清楚；不覆盖原图。

### 阶段三：建立 CNN baseline

任务：提取原作的多层 activation、style descriptor 和 spatial feature。
验收：能够重复运行；输出尺寸和层含义记录完整；至少得到一组可解释的特征图。

### 阶段四：制作受控变体

任务：完成 style-only、geometry-only 和 topology-breaking 版本。
验收：每个版本有输入、参数、随机种子和修改说明；没有不受控制的大量内容漂移。

### 阶段五：比较与分析

任务：计算指标，制作图表，比较原作、变体和毕加索版本。
验收：每个结论都能指向具体图像、feature map 或数值；不把主观判断包装成客观事实。

### 阶段六：创作最终作品

任务：根据实验结果决定保留、改变或破坏哪些元素。
验收：最终作品回答核心问题，而不是单纯追求视觉效果。

### 阶段七：完成叙事与展示

任务：组织个人引言、方法、结果、限制和反思。
验收：不懂 CNN 的观众也能理解“改变了什么、模型看到了什么、我因此学到了什么”。

## 17. 风险与边界

### 17.1 数据量不足

五张图片不能构成可靠训练集。解决方式是使用预训练 CNN 做 feature extraction，并把任何小模型准确称为单作品实验或探索性模型。

### 17.2 变量无法完全分离

改变 style 时也可能改变边缘，改变 geometry 时也可能改变局部纹理。解决方式是记录残余变化，并同时报告 style distance 和 geometry distance，而不是宣称完成了绝对 disentanglement。

### 17.3 生成模型造成内容漂移

生成式 inpainting 可能同时改变人物面孔、服装、色彩和位置。解决方式是使用局部 mask、固定随机种子、结构控制和人工复核；必要时优先采用确定性的几何变换。

几何 warp 还可能引入插值伪影。执行阶段应增加一次零位移的 sham warp：让图像经过完全相同的处理流程但不改变几何，以测量流水线本身造成的特征距离。

### 17.4 CNN 对绘画的领域偏差

很多模型主要在照片上训练，可能误判镜子、画布、暗部和绘画人物。项目不应隐藏这些错误，而应分析这种失败对“机器如何观看艺术”意味着什么。

### 17.5 过度使用神经科学语言

CNN activation 不是人脑活动，feature-space 可视化也不是大脑认知地图的直接证据。项目只能把人工神经网络作为比较工具，研究机器表征和人的解释之间的差异。

### 17.6 技术范围过大

同时完成 style transfer、深度、分割、动画、互动装置和人类实验可能导致每一项都不深入。应先完成最小方案，再按对核心问题的贡献选择扩展项。

### 17.7 图像来源与版权

高清原作、两张几何示意图、局部裁剪和毕加索版本都需要记录原始来源。特别是辅助 PNG 当前缺少完整来源信息，最终提交前需要补全引用。课堂分析与展示不等于可以自由再分发图片。

### 17.8 工程风险

- 高清原图解码后占用大量内存，必须使用代理图或分块；
- 文件名包含组合重音字符，脚本中不宜手写路径；
- 所有生成结果应写入独立输出目录，不能覆盖素材；
- 模型、参数、随机种子和输入版本需要记录，以保证可重复性。

以下问题是制定计划时保留的范围判断。正式执行采用了“预训练 CNN 可接受、提交包含代码/图像/视频/报告/PPT、ready-made 以概念 mockup 呈现、几何同时输出 feature map/边界/线段/候选点、最终作品加受控序列”的完整方案：

1. 老师要求“使用 CNN”是允许使用预训练网络，还是必须亲自训练？
2. 最终交付形式是演示文稿、代码、图片、视频、短文，还是它们的组合？
3. 本学期是否仍然要求 ready-made？
4. “extract geometrical features” 是否必须输出明确的消失点、线条和距离，还是 feature map 可视化即可？
5. 老师期待的是一件最终作品，还是原作与多个 transformation 的分析序列？
6. 是否需要在课堂现场展示模型运行过程？

在得到这些答案前，可以继续细化项目逻辑，但不应过早锁定复杂实现。

## 20. 当前推荐结论

现阶段最强、最符合课程与老师反馈的主线是：

> **使用 CNN 分离并比较《宫娥》的 style、geometry 和 relational topology，通过受控 transformation 检验哪些视觉层次真正让观看者感觉自己进入画中，并把实验结果发展为一件个人作品。**

这个方向保留了原先“神经网络作为不完整观看者”的概念，同时让 CNN、transformation、几何提取、风格提取、topological ideas 和最终创作形成一个可验证的完整闭环。

项目最重要的价值不在于使用了多大的模型，而在于：

- 每个视觉概念都有明确操作性定义；
- 每次 transformation 都有受控变量；
- CNN 输出可以被观察和比较；
- 人类与机器的判断差异能够被讨论；
- 最终作品来自实验结论，而不是随机生成。
